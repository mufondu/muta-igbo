## Reuse Rule

These assets should become a shared MekaOps Edu design asset library.

Future apps should reference this same visual language instead of creating unrelated illustration styles.

## Product Rule

When a new custom illustration is approved for one MekaOps Edu product, consider whether it should become part of the shared library.

## Do Not Use

- Generic emoji as final lesson art
- Procedural placeholder portraits
- Duplicated role images
- Cropped reference boards
- Unlicensed third-party art
- Any reference-only source material as visible app content

## No Generic Icon Rule

MekaOps Edu products should not ship generic emoji/icons as final lesson visuals.

Allowed visual states:

1. Approved custom illustration asset.
2. Clean text badge fallback until the custom illustration exists.

Not allowed as final production UI:

- Generic emoji lesson icons
- Random system emoji
- Placeholder stock icons
- Mixed icon styles
- Duplicated role images
- Cropped sprite sheets
- Embedded text inside illustration assets

All new lesson categories should receive a reusable custom illustration pack before release.
